================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Question: Quality risk still returns 32% after deployment?
Answer: Check terminal logs:
  - If you see "[QUALITY RISK ERROR]" → model inference issue (read traceback)
  - If you see "[BUILD_QUALITY_FEATURES]" but not "[QUALITY RISK] Calling..." → loading issue
  - If you see nothing → feature builder not called (check sprint_context)

Question: Terminal logs are empty?
Answer: Logs go to stderr. Redirect:
  python main.py 2>&1 | grep "\[QUALITY\|\[ERROR\]"

Question: My CI/CD doesn't capture stderr?
Answer: Update CI to capture both stdout and stderr:
  docker logs <container> 2>&1
