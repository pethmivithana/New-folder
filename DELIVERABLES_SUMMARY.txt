COMPLETE SPECIFICATION OF 6 RULES:

Rule 0: Sprint Ending
─────────────────────

THEN action = DEFER

Rule 0.5: Emergency Protocol
────────────────────────────
IF priority in (Critical, Highest)
THEN action = SWAP or OVERLOAD (depending on capacity)
Reasoning: Production bugs bypass all ML checks
Sub-case A: SWAP if candidate exists (lowest priority, fewest SP)
Sub-case B: OVERLOAD if no candidate found (accept overload)

Rule 1: Monster Ticket
──────────────────────
IF story_points >= 13 AND days_remaining < 10
THEN action = SPLIT
Reasoning: Too large to complete in remaining time
Recommendation: Break into 50/50 or phase-based slices

Rule 2: ML Safety Net
─────────────────────
IF ANY of these breached:
  - schedule_risk > threshold (30%-70% depending on risk appetite)
  - productivity_drag < threshold (-20% to -40%)
  - quality_risk > threshold (60%-80%)
THEN action = DEFER
Reasoning: Multiple ML signals indicate high risk

Risk Appetite Thresholds:
  STRICT:   schedule > 30%, drag < -20%, quality > 60%
  STANDARD: schedule > 50%, drag < -30%, quality > 70% [DEFAULT]
  LENIENT:  schedule > 70%, drag < -40%, quality > 80%

Rule 3: Capacity Available
───────────────────────────
IF free_capacity >= story_points AND no ML risks triggered
THEN action = ADD
Reasoning: Capacity exists and all signals are safe

Rule 4: Swap Candidate
──────────────────────
IF free_capacity < story_points AND priority in (High, Critical)
  AND swap_candidate exists (lower priority, lower SP)
THEN action = SWAP
Reasoning: High-value ticket justifies removing low-value item

Rule 5: Fallback
────────────────
IF free_capacity < story_points AND no swap candidate
THEN action = DEFER
Reasoning: Safe default when capacity full and no swap available
